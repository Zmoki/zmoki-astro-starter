Resources show up in the **Resources** sidebar panel and each get their own page at `/resources/{slug}/`.

Use them for things worth linking to on their own: a downloadable file, a guide, a template, a tool you made.

- `type: "page"` renders a full page like this one.
- `type: "link"` is just an outbound link (set the `url` field) — no page is generated.

## Gating a resource (lead magnet)

Attach an optional `form` (Brevo email capture) and `asset` (the deliverable) to gate a
resource behind an email exchange:

- `form` renders a `BrevoForm` above the content. On submit, Brevo emails the asset and
  redirects to `/thank-you/resources/{slug}/`.
- `asset.url` is a **public URL** to the file — host it externally (R2, S3, a CDN/bucket,
  Brevo's own hosting…) rather than committing binaries to this repo. The thank-you page
  shows it as a **direct download** so delivery doesn't depend solely on the email landing.

Note this is a _lead-magnet_ gate, not access control: a public URL is reachable by anyone
who has it. See `src/content.config.ts` for the full frontmatter shape.
